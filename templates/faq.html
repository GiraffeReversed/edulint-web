Why does the browser warn me when I download my code?

The file you download always contains your own code, exactly as you see in the in-browser editor (check the downloadFile function in the page's source code, if you dare). The file should therefore be just as harmful as you yourself make it.

If you keep the file, you can check its contents by opening it in a text editor â€“ this should pose no security risk, even if the file contains potentially harmful code.

The browser warns you agains keeping the downloaded file, because it is an executable Python file. As such, running it could harm your computer. Nevertheless, simply saving the code is safe, but do not forget that double-tapping the file runs it, rather than open it in a text editor.

Why does ctrl + S do nothing in the editor?

A programmer may fall into habit of pressing ctrl + S quite often to avoid the mistake of running unsaved code. Nevertheless, had the page opened the dialog to save the page every time a programmer pressed the shortcut out of a habit, it would be tedious to edit the code directly in the browser.
